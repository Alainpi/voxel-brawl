# Voxel Brawl — MVP 0.0.1 Design Document

**Date:** 2026-03-07
**Goal:** A single-player voxel destruction tech demo that demonstrates the core feel of the game.

---

## Summary

A playable demo where the player walks up to a destructible voxel humanoid dummy and tears it apart using fists and a revolver. Voxels blast off in chunks, limbs detach as physics objects, and debris fills the arena. The dummy auto-resets on death. No menus, no multiplayer, no cards — just the core destruction loop.

All systems are written as production code with clean upgrade paths to the full game.

---

## Approach

**Approach B — Skeleton for Player + Voxel Dummy.**

- Player character: Blender-imported skeleton with walk, idle, punch, shoot animations. Blocky low-poly humanoid, flat-shaded. Not destructible in this MVP.
- Target dummy: Pure voxel humanoid assembled from 6 MagicaVoxel segments. No skeleton. Fully destructible.
- Phase 1 upgrade path: VoxelSegment nodes mount onto the player skeleton's bones. No code changes — just new scene structure.

## Voxel Destruction System

### VoxelSegment Properties

```
voxel_data: Dictionary          # Vector3i → Color
mesh_instance: MeshInstance3D
total_voxel_count: int          # set on load
current_voxel_count: int        # decrements on hits
detach_threshold: float         # ratio at which limb detaches
is_detached: bool
bone_attachment: String         # empty in MVP, used Phase 1
```

### Hit Processing Flow

```
take_hit(hit_pos, radius, damage)
  1. Find all voxels within radius → mark for removal
  2. Remove marked voxels from voxel_data, update count
  3. Spawn debris via DebrisPool
       closest 5–8 voxels → RigidBody3D cubes from pool
       remaining → GPUParticles3D one-shot burst (20–40 particles)
  4. rebuild_mesh() — SurfaceTool pass over remaining voxel_data
  5. Check detach threshold → call detach() if below
```

Batching: multiple hits within one physics frame queue and trigger a single `rebuild_mesh()` call at end of `_physics_process()`.

### SurfaceTool Mesh Generation

For each voxel in `voxel_data`: check 6 neighbors. Emit a quad (with face normal + vertex color) for every exposed face. Flat shading — no smooth normals. No UV mapping, no textures.

### Limb Detachment Thresholds

| Segment | Threshold | Notes |
|---|---|---|
| Head | 30% | Flies off |
| Arm L/R | 30% | Flops away |
| Leg L/R | 40% | Slightly easier to lose |
| Torso | 0% | Cannot detach — triggers dummy death |

On detach: reparent to Main, add RigidBody3D, apply outward + upward impulse. Detached segment retains its voxel Dictionary and remains destructible.

### Debris Pool

- 80 pre-spawned RigidBody3D cubes (0.1×0.1×0.1 units), round-robin recycled
- 1 GPUParticles3D node, re-triggered per hit (one-shot burst)
- Physics cubes auto-return to pool after 3 seconds

### Performance Targets

| Metric | Target |
|---|---|
| Mesh rebuild per segment | < 1ms |
| Active RigidBody3D debris | ≤ 80 |
| GPU particles per hit | 20–40 |
| Framerate | 60fps stable on mid-range laptop |

## Player Controller

**Camera:** Over-the-shoulder, FOV 75°, pitch clamped ±80°. Subtle walk bob. 0.1s camera shake on successful hit.

**Movement:**

| Input | Action |
|---|---|
| WASD | Move relative to camera facing |
| Mouse X | Rotate player (yaw) |
| Mouse Y | Tilt camera (pitch) |
| Space | Jump |
| Shift | Sprint (1.6× speed) |
| ESC | Release mouse / pause |

**Constants:** Speed 5.0 m/s, Jump 6.0, Gravity 9.8, Mouse sensitivity 0.002.

**Animation states:** idle, walk, punch (one-shot), shoot (one-shot) — driven directly from player.gd. No AnimationTree needed for MVP.

**Multiplayer readiness:** Input processing wrapped in `if not is_multiplayer_authority(): return` from day one.

### Revolver (`weapon_ranged.gd`)

| Property | Value |
|---|---|
| Input | LMB |
| Damage | 35 |
| Fire rate | 1 shot / 0.5s |
| Ammo | 6 shots |
| Reload | R key, 1.5s |
| Hit detection | RayCast3D from camera center |
| Voxel removal radius | 2.0 voxels |

Weapon switch: scroll wheel or 1/2 keys.

### Damage Flow (Multiplayer-Ready)

All hits route through `DamageManager` autoload:
```gdscript
func process_hit(segment, hit_pos, radius, damage):
    if multiplayer.is_server():  # always true in single-player MVP
        segment.take_hit(hit_pos, radius, damage)
```
Phase 3: add `@rpc` decorator. Nothing else changes.

## Scene Flow

```
Launch → Load Main scene directly (no menu)
       → Player + Dummy spawn, mouse captured
       → Destroy dummy
       → 1s pause → debris fades → dummy reloads fresh .vox files
       → Ready to destroy again
```

No splash screen, no main menu, no settings for 0.0.1.
